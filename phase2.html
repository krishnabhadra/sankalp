Who's This For

BUILT FOR
UNDERGRADS.

Designed for engineering students ready to move beyond textbooks — into neurorobotics, bio-signal control, haptics, and cyber-physical systems.

This program bridges the gap between foundational academics and real-world robotic systems. Whether you're in Computer Science, Electronics, Mechanical, Biomedical, or AI — if you're curious about how machines can understand and respond to humans, this track is yours.

The format is intentionally progressive: start with a shared online foundation, gain perspective from expert practitioners, and then specialize in hands-on workshops with dedicated mentors.

Workshop Tracks

CHOOSE YOUR
SPECIALIZATION.

Three parallel, mentor-guided tracks — each producing a real, working prototype across 3 intensive offline days.

01
🧠
Neurorobotics & Multimodal Bio-Signal Control
EMG · Vision · Arduino · Assistive Tech

Build an EMG-driven assistive robotic system with vision-assisted interaction — simulating real-world prosthetics and rehabilitation devices.

  • Day 1: EMG signal acquisition, sensor interfacing, and real-time signal visualization
  • Day 2: Noise filtering, adaptive control logic, and vision–EMG multimodal fusion
  • Day 3: End-to-end system testing, performance validation, and deployment demo
Output → EMG-controlled robotic actuator with vision-based supervisory control
02
🤚
Haptics & Tactile Human–Robot Interaction
Force Sensing · Haptic Feedback · Teleoperation

Design a haptic-feedback teleoperation system — translating robotic contact into human sensation for intuitive "blind" control via touch.

  • Day 1: FSR interfacing, pressure calibration, and real-time tactile data visualization
  • Day 2: Haptic rendering with LRA actuators, haptic icons, and latency minimization
  • Day 3: Closed-loop testing, sensory substitution validation, and live demo
Output → Haptic teleoperation system with wearable tactile feedback
03
⚙️
Adaptive Cyber-Physical Systems
IoT · Cloud · Edge AI · Industrial Safety

Build an intelligent safety and monitoring CPS node — sensing, processing at the edge, communicating with the cloud, and responding to safety-critical scenarios.

  • Day 1: Sensor integration, real-time acquisition, and edge-level control logic
  • Day 2: Cloud dashboard, MQTT connectivity, and bi-directional remote control
  • Day 3: Adaptive behavior, fault tolerance, and full end-to-end deployment
Output → Cloud-connected CPS node with adaptive control & real-time monitoring
